# MuMu 红果排行榜自动缓存设计

## 目标

新增一个独立脚本，用于在 MuMu 模拟器中自动操作红果免费短剧 App，进入排行榜并处理前 10 部短剧。脚本只触发 App 内官方“下载/缓存/离线看”能力，依赖 MuMu 共享目录让下载产物在宿主机可见，并整理到 `output/hongguo_videos`。

本设计不包含抓接口、解密缓存、读取 App 私有目录、绕过 DRM 或导出受保护缓存。

## 范围

### 包含

- 连接 MuMu 模拟器上的 Android 设备。
- 启动红果免费短剧 App。
- 导航到排行榜页面。
- 遍历排行榜前 10 条短剧。
- 打开短剧详情页并触发官方缓存/下载入口。
- 轮询 MuMu 共享目录在宿主机上的落盘变化。
- 将可见文件整理到 `output/hongguo_videos\<剧名>\`。
- 写入运行状态、结果汇总和失败调试证据。
- 支持断点续跑和跳过已成功条目。

### 不包含

- 不抓取红果后端 API。
- 不破解、解密或导出 App 私有缓存。
- 不绕过会员、权限、地域、版权或平台限制。
- 不保证红果官方缓存一定写入 MuMu 共享目录。
- 不内置验证码、登录风控或反自动化绕过能力。

## 推荐方案

采用 `adb + uiautomator2` 为主、少量坐标兜底的混合方案。

纯坐标方案实现快但稳定性差，分辨率、窗口缩放和 App 小版本变化都会导致失败。Appium 方案结构完整但搭建和维护成本过高，不适合当前脚本型仓库。混合方案能用控件文本和资源标识处理稳定步骤，用坐标覆盖排行榜卡片这类可能没有清晰控件标识的区域。

## 文件布局

建议新增：

- `scripts/mumu_hongguo_downloader.py`：主脚本。
- `scripts/hongguo_downloader_config.json`：配置文件。
- `output/hongguo_download_state.json`：断点状态。
- `output/hongguo_download_run.json`：运行结果。
- `output/hongguo_download_debug/`：失败截图、页面层级和调试日志。

最终下载整理目录：

- `output/hongguo_videos\<剧名>\`

## 主要模块

### `adb_client`

负责基础设备能力：

- 检查 `adb` 是否可用。
- 连接 MuMu 设备。
- 获取设备列表和当前前台 App。
- 执行点击、滑动、返回、Home、截图。
- 启动红果 App 包名和 Activity。

### `ui_session`

负责 UI 自动化封装：

- 初始化 `uiautomator2`。
- 按文本、描述、resource-id 查找控件。
- 等待控件出现或消失。
- 提供点击、输入、滑动列表等高级动作。
- 找不到控件时调用配置中的坐标兜底。

### `rank_navigator`

负责排行榜导航：

- 打开红果首页。
- 进入排行榜页面。
- 确认当前处于排行榜。
- 逐个打开前 10 名条目。
- 从详情页返回排行榜并恢复当前位置。
- 记录排名和剧名。

### `download_controller`

负责下载触发：

- 在详情页查找 `下载`、`缓存`、`离线看` 等官方入口。
- 判断是否已下载、是否正在下载、是否需要确认弹窗。
- 点击下载后等待 App 内状态变化。
- 超时或控件缺失时保存失败证据。

### `shared_folder_watcher`

负责宿主机文件检测：

- 读取 MuMu 共享目录对应的宿主机路径。
- 在下载触发前记录目录快照。
- 触发下载后轮询新增或变化文件。
- 过滤常见视频扩展名和临时文件。
- 将稳定后的文件移动或复制到 `output/hongguo_videos\<剧名>\`。

### `run_state`

负责状态持久化：

- 记录每个排名的剧名、状态、输出路径、错误信息和重试次数。
- 支持中断后续跑。
- 对已成功条目默认跳过。
- 对未知或失败条目允许重试。

## 配置文件

`scripts/hongguo_downloader_config.json` 应包含：

```json
{
  "adb_serial": "",
  "adb_connect": "",
  "app_package": "",
  "app_activity": "",
  "shared_folder": "",
  "output_dir": "E:\\playletvideo",
  "top_n": 10,
  "resume": true,
  "skip_success": true,
  "timeouts": {
    "app_launch": 30,
    "page_wait": 20,
    "download_start": 30,
    "file_stable": 300
  },
  "keywords": {
    "rank": ["排行榜", "热榜", "榜单"],
    "download": ["下载", "缓存", "离线看", "离线缓存"],
    "confirm": ["确定", "确认", "继续"]
  },
  "fallback_taps": {
    "rank_tab": null,
    "first_rank_item": null,
    "download_button": null,
    "back_button": null
  }
}
```

`app_package` 和 `app_activity` 可以通过 `adb shell dumpsys window` 或包管理命令校准后填入。`fallback_taps` 为可选坐标，格式建议为 `[x, y]`。

## 执行流程

1. 检查 `adb`、MuMu 设备、`uiautomator2`、共享目录和输出目录。
2. 启动红果 App。
3. 导航到排行榜页面。
4. 从第 1 名开始遍历到第 10 名。
5. 打开当前短剧详情页。
6. 读取剧名并写入状态。
7. 触发 App 内官方下载/缓存入口。
8. 轮询共享目录，等待新增文件稳定。
9. 将文件整理到 `output/hongguo_videos\<剧名>\`。
10. 标记当前条目成功、跳过或失败。
11. 返回排行榜并继续下一条。
12. 写入最终运行汇总。

## 错误处理

- 关键页面未出现时，最多重试 2 到 3 次。
- 进入详情失败时，返回排行榜并重新点击当前条目。
- 下载入口不存在时，保存截图和 UI 层级，标记当前条目失败。
- 共享目录没有新增文件时，标记为“已触发下载但未确认落盘”。
- 当前条目失败不阻塞后续条目。
- 每次失败都记录错误原因、当前排名、剧名、截图路径和 UI XML 路径。

## 断点续跑

状态文件以“排名 + 剧名”为任务键。

- `success`：默认跳过。
- `failed`：默认允许重试。
- `download_triggered`：优先检查目标目录和共享目录是否已有新文件，再决定是否重新触发。
- `skipped`：保留跳过原因，不重复处理。

## 校准模式

脚本应提供一个轻量校准模式，用于首次运行或 App 改版后更新配置。

校准模式建议执行：

- 检查设备连接。
- 输出当前前台包名和 Activity。
- 保存当前截图。
- 保存当前 UI 层级。
- 允许用户手工打开排行榜后，记录排行榜入口、第一条卡片、下载按钮、返回按钮等兜底坐标。

主流程优先使用控件选择器，坐标只作为兜底。

## 测试方案

### 连通性测试

验证：

- `adb` 可用。
- MuMu 设备在线。
- `uiautomator2` 能连接。
- 可以截图。
- 可以读取共享目录。
- 可以创建 `output/hongguo_videos`。

### 单条流程测试

只处理排行榜第 1 条，验证：

- 能进入排行榜。
- 能进入详情页。
- 能触发官方下载/缓存。
- 共享目录能检测到新文件。
- 文件能归档到目标目录。

### 批量流程测试

处理前 10 条，验证：

- 失败条目不阻塞后续条目。
- 断点续跑可用。
- 已成功条目不会重复下载。
- 汇总 JSON 清晰可读。
- 调试截图和 UI 层级完整。

## 验收标准

- 脚本能自动连接 MuMu 并启动红果 App。
- 能进入排行榜并处理前 10 个条目。
- 每个条目都有明确状态：`success`、`skipped` 或 `failed`。
- 成功条目在 `output/hongguo_videos\<剧名>\` 下出现宿主机可见文件。
- 失败条目保存截图、UI 层级和错误原因。
- 脚本中断后再次运行可以续跑。
- 脚本不读取 App 私有缓存，不抓接口，不做 DRM 绕过。

## 风险与边界

- 如果红果官方缓存只写入 App 私有目录，而不是 MuMu 共享目录，脚本无法把视频整理到 `output/hongguo_videos`。
- 如果 App 版本变动导致控件文本变化，需要更新配置中的关键词或坐标。
- 如果登录、验证码、风控、权限弹窗阻塞流程，脚本只保存证据，不尝试绕过。
- 如果排行榜页面需要人工选择频道或分类，应先通过配置或校准模式固定入口。
